# Storybook

O **Storybook** é uma ferramenta poderosa para desenvolvimento de componentes de interface do usuário. Neste projeto, utilizamos o Storybook para React Native para documentar, testar e visualizar nossos componentes de forma isolada.

## Estrutura do Storybook

Nosso projeto utiliza o Storybook para criar histórias (stories) dos componentes, permitindo:

1. Visualizar diferentes estados de um componente
2. Documentar as propriedades de cada componente
3. Testar interações e comportamentos
4. Fornecer exemplos de uso para desenvolvedores

## Como Usar o Storybook

Para iniciar o Storybook:

```bash
yarn storybook:generate
yarn android # ou yarn ios
```

## Criando Histórias

Cada componente deve ter seu próprio arquivo de histórias seguindo a convenção `.stories.tsx`. Exemplo:

```tsx
// src/components/Button/index.stories.tsx
import type { Meta } from '@storybook/react';
import Button from '.';

const meta: Meta<typeof Button> = {
  title: 'Button',
  component: Button,
  argTypes: {
    onPress: { action: 'clicked' },
    // outras propriedades
  },
  args: {
    // valores padrão
    children: 'Botão Exemplo',
  },
  parameters: {
    notes: 'Documentação do componente Button',
  },
};

export default meta;

// Histórias diferentes
export const Default = {};

export const Primary = {
  args: {
    colorVariant: 'primary',
  },
};

export const Secondary = {
  args: {
    colorVariant: 'secondary',
  },
};
```

## Integração com a Documentação

A integração entre o Storybook e nossa documentação em Rspress permite:

1. **Consistência**: Garantir que os exemplos na documentação reflitam comportamentos reais
2. **Verificação**: Facilitar a verificação visual de que os componentes estão funcionando conforme o esperado
3. **Desenvolvimento**: Permitir que desenvolvedores testem alterações em componentes de forma isolada

## Boas Práticas

1. **Crie histórias para todos os estados**: Documente os diferentes estados possíveis de cada componente
2. **Adicione notas explicativas**: Use o campo `notes` para fornecer contexto e informações importantes
3. **Use controles**: Configure `argTypes` para permitir que usuários manipulem as propriedades do componente
4. **Documente acessibilidade**: Inclua exemplos de uso com suporte a acessibilidade
